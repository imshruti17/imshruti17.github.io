Simple Event-Driven Image Processing Application

The pipeline uses two S3 buckets, a source bucket and a processed bucket. When images are added to the source bucket a lambda function is triggered based on the PUT. When invoked the lambda function receives the event and extracts the bucket and object information. Once those details are known, the lambda function, using the PIL module pixelates the image with 5 different variations (8x8, 16x16, 32x32, 48x48 and 64x64) and uploads them to the processed bucket.

Serverless Reminder App

The application loads from an S3 bucket and run in browser, communicating with Lambda and Step functions via an API Gateway Endpoint. Using the application will be able to configure reminders for 'pet cuddles' to be sent using email.

Simple WEB Identity Federation App

A simple serverless application which uses Web Identity Federation. The application runs using the technologies such as S3 for front-end application hosting, Google API Project as an ID Provider, Cognito and IAM Roles to swap Google Token for AWS credentials. The application runs from a browser, gets the user to login using a Google ID and then loads all images from a private S3 bucket into a browser using presignedURLs.
